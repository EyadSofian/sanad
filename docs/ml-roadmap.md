# خطة الموديلات — من الـ APIs لموديل داخلي

> إجابة عملية على سؤال: «عايز أدرب موديل داخلي (ML) أو أدمج بين LLM ولانج تشين أو موديلات متدربة داخليًا»

## الخلاصة في سطرين

دلوقتي (MVP): **مفيش تدريب داخلي** — الـ abstraction بتاعة المحركات (`server/src/lib/llm.js`) جاهزة وبتديك Gemini / GPT / Claude بتغيير env var واحد.
بعدين: كل رسالة بتعدي في سند بتبني **الداتاسيت** اللي هتدرّب عليه — أول موديل داخلي منطقي هو **الـ Router**، وآخر مرحلة هي **شخصية سند نفسها** بـ LoRA. LangChain مش محتاجينها.

---

## المرحلة 0 — دلوقتي (MVP / البيتا)

- المحرك الأساسي switchable: `LLM_PROVIDER=gemini|openai|anthropic`.
- الـ Router والـ embeddings وتحويل الصوت ثابتين على Gemini (رخيصين وسريعين، وطبقة الأمان لازم تفضل deterministic).
- **ليه مفيش تدريب دلوقتي؟** ٣ أسباب: مفيش داتا كفاية، التكلفة (تدريب + hosting + صيانة)، وأي موديل داخلي لازم يعدي نفس اختبارات الأمان (crisis suite + diagnosis guard) قبل ما يلمس مستخدم.

## المرحلة 1 — جمع الداتا (أثناء البيتا، شغّالة لوحدها تقريبًا)

الداتا بتتسجل أصلًا في الـ schema الحالي:

| جدول | إيه اللي فيه | هيدرّب إيه |
|---|---|---|
| `messages` | كل رسالة + `persona` + `emotion` + `intensity` (تصنيف flash-lite) | الـ Router الداخلي |
| `sessions.summary` | ملخصات المحادثات | جودة الردود / التقييم |
| `memory_facts` | حقائق مستخلصة | fact-extraction الداخلي |
| `crisis_events` | كل تفعيل لبروتوكول الأمان | تقوية الـ pre-check |

المطلوب إضافته قبل التدريب: بند موافقة صريح في الـ Privacy («بياناتك ممكن تُستخدم مجهولة الهوية لتحسين النموذج») + سكريبت anonymization (شيل الإيميلات/الأسماء/الأرقام) + زرار 👍/👎 على الردود (label رخيص وقيّم جدًا).

## المرحلة 2 — أول موديل داخلي: الـ Router (أعلى عائد، أقل مخاطرة)

الـ Router بيتنده **في كل رسالة** — ده أعلى volume وأبسط مهمة (تصنيف، مش توليد):

- **الطريقة:** distillation — الـ labels اللي flash-lite كتبها في `messages` هي الداتاسيت الجاهز.
- **الموديل:** fine-tune لموديل عربي صغير (CAMeLBERT / AraBERT / MARBERT) أو SetFit لو الداتا قليلة (~2k رسالة تكفي بداية).
- **الناتج:** classifier بيطلع نفس الـ JSON (`persona/emotion/intensity/crisis/lang`) بـ latency أقل من 50ms وتكلفة صفر تقريبًا.
- **التركيب:** نفس الـ interface بتاع `routerService.js` — بيتحط وراه flag `ROUTER_PROVIDER=internal` والـ regex pre-check شغال قبله زي ما هو (مبيتلمسش أبدًا).
- **شرط الإطلاق:** يعدي نفس الـ CI suites + recall على `crisis=true` أعلى من أو يساوي flash-lite. لو أقل بأي نسبة → مايتنشرش.

## المرحلة 3 — شخصية سند (LoRA على open-weights)

لما يبقى عندك 5–10k محادثة كويسة ومُقيّمة:

- fine-tune بـ **LoRA/QLoRA** على موديل مفتوح (Qwen2.5-7B أو Llama-3.1-8B — الاتنين كويسين في العربي) على أحسن المحادثات (اللي واخدة 👍 وعدّت المراجعة).
- الهدف: نبرة سند المصرية الطبيعية من غير ما تدفع لـ API خارجي في كل رسالة، مع خصوصية أعلى (الداتا مبتخرجش).
- hosting: vLLM على GPU instance أو Hugging Face Inference Endpoints.
- **وبيتركب كـ provider رابع في `llm.js`** — نفس الـ abstraction الموجودة دلوقتي، من غير ما تلمس باقي الكود. ده بالظبط ليه بنيناها كده.
- بديل أسرع قبل الـ LoRA: **Gemini tuned models** (fine-tuning مُدار من جوجل) — أقل شغل بس أقل تحكم.

## ليه مش LangChain؟

- الـ orchestration بتاعتنا (router → context → personas → safety) **مكتوبة مخصوص وأخف وأوضح** — LangChain هيضيف طبقة abstractions إحنا مش محتاجينها وهيصعّب الـ debugging.
- طبقة الأمان لازم تفضل **deterministic ومقروءة سطر سطر** — ده عكس فلسفة الـ chains الجاهزة.
- لو احتجنا RAG أعقد للـ KB (مصادر كتير + re-ranking) نبص ساعتها على LlamaIndex أو LangGraph **كأداة للجزئية دي بس**، مش استبدال للنظام.

## قاعدة ثابتة لأي موديل داخلي

أي موديل داخلي — router أو شخصية — لازم يعدي **نفس بوابات الأمان** اللي في CI (crisis suite الـ 60 جملة + diagnosis guard الـ 15 سؤال) قبل ما يوصل لأي مستخدم. السيفتي مش بتتبدل مع المحرك.
